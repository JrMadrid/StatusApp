# COMPONENTS

Este directorio alberga los componentes reutilizables de la interfaz de usuario que se utilizan en diversas partes de la aplicación.​

Propósito
- Centralizar elementos de UI reutilizables como pantallas, formularios, tarjetas, paneles, entre otros.
- Promover la reutilización de código y mantener una interfaz de usuario coherente en toda la aplicación.​

/Data
Alberga los componentes para usuarios de menor nivel.

/Elements
Elementos de código que conforman la interfaz.

/Informativas
Componentes de páginas informativas.

/Listas
Botones para guardar los datos de las tablas como documentos PDF o Excel.

/Panel_
Componentes que conforman los CRUD correspondientes de cada tabla.

/PDF
Formatos de datos para guardarlos en PDF.

Características
- Cada componente se encuentra en su propio directorio, conteniendo archivos como .jsx.
- Se pueden organizar en subcarpetas según su función o nivel de complejidad.​

Notas
- Nombrar los componentes utilizando Snake_Case para mantener la coherencia y facilitar su identificación.
- Incluir documentación o comentarios que describan la funcionalidad y props esperadas de cada componente.
- Evitar incluir lógica de negocio compleja dentro de los componentes; en su lugar, delegar dicha lógica a hooks o servicios externos.
