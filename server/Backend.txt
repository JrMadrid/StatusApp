# Server

Este directorio contiene todo el backend del proyecto.
Se encarga de gestionar las conexiones a dispositivos, la base de datos, y el procesamiento de datos.

Estructura principal
- config/: Configuración de variables de entorno.
- connection/: Conexiones a dispositivos UPS, iLO, biométricos y validaciones de red.
- controllers/: Lógica para manejar las solicitudes y respuestas.
- db/: Configuración y conexión a la base de datos, incluyendo manejo de sesiones.
- models/: Modelado y validación de los datos.
- routes/: Definición de rutas para acceder a la API.
- service/: Extracción y procesamiento de datos de los dispositivos.
- server.js: Archivo principal para inicializar el servidor.

Tecnologías utilizadas
- Node.js
- Express.js
- SQL (Base de datos de sesiones)
- SSH / TCP/IP (Para conexión con dispositivos)
- dotenv (Manejo de variables de entorno)
